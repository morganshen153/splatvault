# Distribution Plan

`SplatVault` uses a three-layer distribution model.

## Layer 1: Public Repository

Goal:

- attract users
- build trust
- let people evaluate the Community Edition

Repository:

- `product-splatvault`

Contains:

- public docs
- Community Edition code
- roadmap
- demo assets
- public issue tracking

## Layer 2: Private Source Repository

Goal:

- protect commercial value
- ship Pro features
- control release quality

Repository:

- `private-pro-splatvault`

Contains:

- Pro features
- commercial source
- private build logic
- private release rules

## Layer 3: Binary Release Channel

Goal:

- deliver software without handing out the whole source

Artifacts:

- Windows installer
- portable zip
- signed update package

## Operator Rule

- public repo is for discovery
- private repo is for commercial source
- binary release is for paid delivery
