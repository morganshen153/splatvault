# Community vs Pro

This document defines the intended product split for `SplatVault`.

The goal is to make the public repository useful, while preventing the full commercial value from being given away for free.

## Public Community Edition

The public repository is intended to provide:

- local-first single-user setup
- image, video, and text asset import foundations
- local previews and thumbnails
- basic search workspace
- project, collection, and tag structure
- transparent roadmap and public issue tracking

The Community Edition is meant to help users:

- evaluate the product direction
- adopt the local-first workflow
- test on their own asset libraries
- follow development in public

## Pro Direction

The paid commercial direction is intended to include:

- production-friendly packaging
- faster indexing workflows
- stronger video retrieval workflows
- advanced export and review flows
- better operator tooling
- license-backed commercial usage rights

The Pro layer is where direct commercial value should accumulate over time.

## Commercial Boundary

The public repository is source-visible, but it is not meant to be a free commercial giveaway.

Commercial production use requires a separate license from the authors.

This is the core rule behind the current repository licensing strategy:

- public code visibility for discovery and trust
- paid rights for commercial production use

## Why This Split Exists

This split supports the project strategy already chosen:

- GitHub-led discovery
- paid open source / open-core style distribution
- reduced dependence on meeting-heavy sales
- stronger protection against direct white-label reuse

## Practical Buyer Message

The public repository is for:

- evaluation
- community follow-up
- technical trust building

The paid path is for:

- commercial usage rights
- stronger workflows
- smoother packaging
- long-term product support
