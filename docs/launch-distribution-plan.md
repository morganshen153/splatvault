# Launch Distribution Plan

## Goal

Drive qualified traffic to the SplatVault public preview without framing it as a finished enterprise product.

## Best Channels

### 1. GitHub

Use for:

- discovery
- trust
- source review
- release downloads

Post:

- README
- release notes
- release assets
- screenshots

### 2. Product Hunt

Best for:

- maker / indie / early adopter discovery
- preview-style launches
- product positioning feedback

Angle:

- local-first multimodal asset search
- public preview
- 3D / video / scan / AI data workflows

### 3. Hacker News

Best for:

- technical credibility
- builder feedback
- honest discussion about local-first workflows

Angle:

- "Show HN: local-first asset search for image/video/scan libraries"

### 4. Reddit

Best for:

- workflow pain discovery
- early adopters
- practical feedback

Highest-fit communities:

- `r/3DScanning`
- `r/photogrammetry`
- `r/vfx`
- `r/LocalLLaMA`
- `r/Database`

### 5. LinkedIn

Best for:

- studio owners
- pipeline leads
- technical artists
- AI/data teams

Angle:

- productivity
- asset reuse
- local-first organization
- preview release

### 6. X / Twitter

Best for:

- quick launch visibility
- repeat exposure
- developer audience

Angle:

- short, direct, problem-first

## Messaging Rules

- Always say `Public Preview`.
- Always say `local-first`.
- Avoid saying `enterprise-ready`.
- Avoid saying `renderer`.
- Avoid overpromising semantic quality.

## Posting Order

1. GitHub release
2. X announcement
3. Reddit community posts
4. LinkedIn post
5. Product Hunt launch
6. Hacker News Show HN

## Best Hook

`Search your local asset library by meaning, not filenames.`

## Suggested Launch Window

- Day 1: GitHub + X
- Day 2: Reddit
- Day 3: LinkedIn
- Day 4: Product Hunt
- Day 5: Hacker News

## What To Watch

- clicks to GitHub
- release downloads
- stars
- comments asking for:
  - better search quality
  - easier install
  - better video retrieval
  - export/review flows
