# Public vs Private Repos

## Public Repo

Name:

- `product-splatvault`

Purpose:

- discovery
- trust
- evaluation
- community visibility

Should contain:

- public documentation
- community feature set
- install instructions
- roadmap
- public issue tracker

Should not contain:

- commercial source
- Pro features
- private build scripts
- private release keys

## Private Repo

Name:

- `private-pro-splatvault`

Purpose:

- commercial source control
- paid development
- private packaging

Should contain:

- Pro features
- license-gated code
- private automation
- release and signing scripts

## Binary Release

Purpose:

- deliver runnable builds to paid users

Examples:

- installer
- portable zip
- signed update package

## Rule

The public repo is for visibility.
The private repo is for value.
The binary is for delivery.
