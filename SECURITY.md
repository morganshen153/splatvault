# Security Policy

## Supported Versions

Security support follows the current public release line.

## Reporting a Vulnerability

Please report security issues privately through the repository owner contact or GitHub security advisories.

Do not open a public issue for sensitive security reports.

## Scope

This repository is intended to be source-visible for evaluation and community use.

Commercial production use, private build pipelines, and premium release tooling are handled separately from the public repository strategy.
